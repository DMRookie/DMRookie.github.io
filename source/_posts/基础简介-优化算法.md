神经网络的训练本质上是一个**优化问题**：我们希望找到一组参数，使得损失函数尽可能小。上一篇文章介绍了梯度下降的基本思想：沿着梯度方向更新参数。但实际训练中，简单的梯度下降往往会遇到各种问题。本文将用一个统一的框架，带你理解各种优化算法是如何解决这些问题的。

## 从下山 analogy 开始

让我们延续上一篇的类比。想象你在山上，想尽快下到谷底。

**梯度下降**的策略是：每走一步，看看哪个方向最陡，就往那个方向走。

但实际下山时，你会遇到几个问题：

1. **步子太大**：可能会越过谷底，甚至越走越高
2. **步子太小**：下山太慢，天黑了还没下完
3. **忽左忽右**：如果谷底在一条窄沟里，你可能在两边反复横跳
4. **陷入局部最优**：可能下到一个坑里，以为到底了，其实旁边还有更深的谷

优化算法就是为了解决这些问题而设计的。它们的核心思路可以归纳为一个统一框架：

$$v_t = \beta \cdot v_{t-1} + g_t$$
$$\theta_t = \theta_{t-1} - \eta \cdot m_t \cdot \frac{v_t}{\sqrt{s_t} + \epsilon}$$

这个框架包含三个核心组件：
- **动量 $v_t$**：历史梯度的累积，帮助我们"冲过"局部起伏
- **自适应学习率 $s_t$**：根据历史梯度调整每个参数的学习率
- **一阶矩估计 $m_t$**：对梯度的平滑估计

不同的优化算法只是对这三个组件的不同实现。让我们逐一拆解。

## 框架组件1：动量（Momentum）

### 问题：梯度噪声导致训练不稳定

在实际训练中，每一步的梯度 $g_t$ 会因为 mini-batch 的随机性而波动。如果没有动量，参数更新会像醉汉走路一样忽左忽右。

![梯度噪声](/images/基础简介-优化算法/01_gradient_noise.png)

### 解决方案：累积历史梯度

动量的思想是：**不要只看当前一步的梯度，而是看历史梯度的累积方向**。

$$v_t = \beta \cdot v_{t-1} + g_t$$
$$\theta_t = \theta_{t-1} - \eta \cdot v_t$$

其中 $\beta$ 是动量系数（通常设为0.9），$v_t$ 是累积的速度。

**直观理解**：把参数更新想象成一个滚动的球。球在下坡时会越来越快，即使路上有小石头（噪声梯度），球也会因为惯性继续向前。这帮助我们：
- 在梯度方向一致的区域加速
- 在梯度方向变化的区域平滑波动

![动量](/images/基础简介-优化算法/02_momentum.png)

**代码实现**：

```python
class SGD_Momentum:
    def __init__(self, params, lr=0.01, momentum=0.9):
        self.params = params
        self.lr = lr
        self.momentum = momentum
        self.v = [0] * len(params)  # 速度

    def step(self):
        for i, param in enumerate(self.params):
            # 更新速度：累积历史梯度
            self.v[i] = self.momentum * self.v[i] + param.grad
            # 用速度更新参数
            param.data -= self.lr * self.v[i]
```

### Nesterov 动量：看得更远

普通动量先计算梯度，再更新参数。但 Nesterov 动量更聪明：**先预估下一步的位置，再在那个位置计算梯度**。

$$\theta_{\text{lookahead}} = \theta_{t-1} + \beta \cdot v_{t-1}$$
$$v_t = \beta \cdot v_{t-1} + g(\theta_{\text{lookahead}})$$
$$\theta_t = \theta_{t-1} - \eta \cdot v_t$$

**直观理解**：普通动量是"先看路再走"，Nesterov 是"先走一步再看路"。这样能更早发现前方的障碍，做出更准确的调整。

## 框架组件2：自适应学习率（Adaptive Learning Rate）

### 问题：不同参数需要不同的学习率

在神经网络中，有些参数经常收到大的梯度，有些参数的梯度很小。如果我们对所有参数用相同的学习率：
- 梯度大的参数：可能更新太猛，震荡
- 梯度小的参数：可能更新太慢，学不动

### 解决方案：根据历史梯度调整学习率

**AdaGrad**的思想是：**对经常收到大梯度的参数降低学习率，对经常收到小梯度的参数提高学习率**。

$$s_t = s_{t-1} + g_t^2$$
$$\theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{s_t} + \epsilon} \cdot g_t$$

其中 $s_t$ 是历史梯度的平方和，$\epsilon$ 是防止除零的小常数。

**直观理解**：想象你在沙地上走路，每走一步都会留下脚印（梯度平方）。走过很多次的地方（梯度大的参数）脚印深，走起来更吃力（学习率降低）；没怎么走过的地方（梯度小的参数）脚印浅，走起来轻松（学习率提高）。

![AdaGrad](/images/基础简介-优化算法/03_adagrad.png)

**代码实现**：

```python
class AdaGrad:
    def __init__(self, params, lr=0.01, eps=1e-8):
        self.params = params
        self.lr = lr
        self.eps = eps
        self.s = [0] * len(params)  # 累积梯度平方

    def step(self):
        for i, param in enumerate(self.params):
            g = param.grad
            # 累积梯度平方
            self.s[i] += g * g
            # 自适应学习率
            param.data -= self.lr / (np.sqrt(self.s[i]) + self.eps) * g
```

**AdaGrad 的问题**：$s_t$ 只增不减，导致学习率越来越小，最终趋近于0，训练提前"刹车"。

### RMSProp：指数移动平均

**RMSProp**解决了AdaGrad学习率衰减过快的问题。它用**指数移动平均**代替累积和：

$$s_t = \beta \cdot s_{t-1} + (1-\beta) \cdot g_t^2$$
$$\theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{s_t} + \epsilon} \cdot g_t$$

其中 $\beta$ 通常设为0.999。

**直观理解**：AdaGrad记住所有历史，RMSProp只记住最近的信息。就像短期记忆vs长期记忆。

## 框架组件3：一阶矩估计（First Moment Estimation）

### 问题：梯度的噪声

即使有了自适应学习率，梯度本身仍然有噪声。我们需要对梯度做一个平滑处理。

### 解决方案：对梯度做指数移动平均

$$m_t = \beta_1 \cdot m_{t-1} + (1-\beta_1) \cdot g_t$$

其中 $\beta_1$ 通常设为0.9。

**直观理解**：就像天气预报用过去几天的平均温度来预测今天，而不是只看昨天。

## 终极形态：Adam

**Adam**（Adaptive Moment Estimation）把前面所有组件组合在一起：

**动量（一阶矩）**：
$$m_t = \beta_1 \cdot m_{t-1} + (1-\beta_1) \cdot g_t$$

**自适应学习率（二阶矩）**：
$$s_t = \beta_2 \cdot s_{t-1} + (1-\beta_2) \cdot g_t^2$$

**偏差修正**：
$$\hat{m}_t = \frac{m_t}{1 - \beta_1^t}$$
$$\hat{s}_t = \frac{s_t}{1 - \beta_2^t}$$

**参数更新**：
$$\theta_t = \theta_{t-1} - \frac{\eta \cdot \hat{m}_t}{\sqrt{\hat{s}_t} + \epsilon}$$

**为什么需要偏差修正？** 训练初期，$m_t$和$s_t$都是从0开始的，会被低估。偏差修正是一个补偿，让初期的估计更准确。

**代码实现**：

```python
class Adam:
    def __init__(self, params, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8):
        self.params = params
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.eps = eps
        self.m = [0] * len(params)  # 一阶矩
        self.s = [0] * len(params)  # 二阶矩
        self.t = 0

    def step(self):
        self.t += 1
        for i, param in enumerate(self.params):
            g = param.grad
            # 更新一阶矩（动量）
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * g
            # 更新二阶矩（自适应学习率）
            self.s[i] = self.beta2 * self.s[i] + (1 - self.beta2) * g * g
            # 偏差修正
            m_hat = self.m[i] / (1 - self.beta1 ** self.t)
            s_hat = self.s[i] / (1 - self.beta2 ** self.t)
            # 参数更新
            param.data -= self.lr * m_hat / (np.sqrt(s_hat) + self.eps)
```

## 统一框架：所有优化算法一览

现在我们可以用一个表格来总结所有优化算法在统一框架下的实现：

| 算法 | 动量 $v_t$ | 自适应学习率 $s_t$ | 一阶矩 $m_t$ | 特点 |
|:-----|:-----------|:-------------------|:-------------|:-----|
| **SGD** | ❌ | ❌ | ❌ | 最简单，只用当前梯度 |
| **SGD+Momentum** | ✅ $\beta v_{t-1} + g_t$ | ❌ | ❌ | 平滑梯度，加速收敛 |
| **Nesterov** | ✅ 预估位置计算梯度 | ❌ | ❌ | 更准确的动量 |
| **AdaGrad** | ❌ | ✅ $s_{t-1} + g_t^2$ | ❌ | 自适应，但学习率衰减快 |
| **RMSProp** | ❌ | ✅ $\beta s_{t-1} + (1-\beta)g_t^2$ | ❌ | 解决学习率衰减问题 |
| **Adam** | ❌ | ✅ $\beta_2 s_{t-1} + (1-\beta_2)g_t^2$ | ✅ $\beta_1 m_{t-1} + (1-\beta_1)g_t$ | 一阶矩+二阶矩，最常用 |
| **AdamW** | ❌ | ✅ | ✅ + 权重衰减解耦 | 解决Adam的权重衰减问题 |

## Adam 的问题与改进

### 问题1：权重衰减的"欺骗"

Adam中使用L2正则化（权重衰减）时，存在一个问题：权重衰减会"欺骗"自适应学习率。

**原因**：权重衰减会改变梯度的大小，但自适应学习率是根据梯度大小来调整学习率的。这导致权重衰减的效果被学习率的调整所抵消。

**解决方案：AdamW**

AdamW将权重衰减从梯度计算中解耦出来，直接在参数更新后应用：

```python
class AdamW:
    def __init__(self, params, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8, weight_decay=0.01):
        self.params = params
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.eps = eps
        self.weight_decay = weight_decay
        self.m = [0] * len(params)
        self.s = [0] * len(params)
        self.t = 0

    def step(self):
        self.t += 1
        for i, param in enumerate(self.params):
            g = param.grad
            # 更新一阶矩
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * g
            # 更新二阶矩
            self.s[i] = self.beta2 * self.s[i] + (1 - self.beta2) * g * g
            # 偏差修正
            m_hat = self.m[i] / (1 - self.beta1 ** self.t)
            s_hat = self.s[i] / (1 - self.beta2 ** self.t)
            # 参数更新（不包含权重衰减）
            param.data -= self.lr * m_hat / (np.sqrt(s_hat) + self.eps)
            # 单独应用权重衰减
            param.data -= self.lr * self.weight_decay * param.data
```

**效果**：AdamW在大模型训练中已成为标配，收敛更快，泛化更好。

### 问题2：大模型训练的稳定性

在大模型训练中，Adam还面临一些挑战：
- **内存占用大**：每个参数需要存储$m_t$和$s_t$，内存开销是SGD的3倍
- **学习率调度敏感**：warmup和cosine decay的参数需要仔细调优
- **泛化gap**：有时简单的SGD+Momentum反而泛化更好

**解决方案**：
- **Muon**：针对大模型设计的优化器，使用模型更新而非梯度更新
- **Adafactor**：减少内存占用的自适应优化器
- **学习率warmup**：训练初期用小学习率，避免参数更新过大

## 实践建议

### 如何选择优化算法？

| 场景 | 推荐算法 | 理由 |
|:-----|:---------|:-----|
| **通用默认选择** | AdamW | 平衡了收敛速度和泛化性能 |
| **大模型预训练** | AdamW + warmup + cosine decay | 工业界标准配置 |
| **需要更好泛化** | SGD+Momentum | 简单，泛化gap小 |
| **内存受限** | Adafactor /Muon | 减少内存占用 |
| **推荐系统** | 稀疏优化器（如FTRL） | 适合稀疏特征 |

### 超参数设置建议

**Adam/AdamW**：
- 学习率 $\eta$：1e-4 ~ 1e-3（大模型常用1e-4）
- $\beta_1$：0.9（一阶矩衰减）
- $\beta_2$：0.999（二阶矩衰减）
- $\epsilon$：1e-8
- 权重衰减：0.01 ~ 0.1

**SGD+Momentum**：
- 学习率 $\eta$：0.01 ~ 0.1
- 动量 $\beta$：0.9
- 学习率调度：余弦退火

### 学习率调度策略

**Warmup + Cosine Decay**：

```python
import math

def get_lr(step, warmup_steps, total_steps, base_lr, min_lr):
    if step < warmup_steps:
        # warmup阶段：线性增长
        return base_lr * step / warmup_steps
    else:
        # cosine decay阶段：余弦衰减
        decay_ratio = (step - warmup_steps) / (total_steps - warmup_steps)
        coeff = 0.5 * (1 + math.cos(math.pi * decay_ratio))
        return min_lr + coeff * (base_lr - min_lr)
```

**为什么需要warmup？** 训练初期，参数随机初始化，梯度噪声大。如果一开始用大学习率，参数更新会很不稳定。warmup让学习率从小值逐渐增大，等训练稳定后再加速学习。

## 总结

通过本文，我们用一个统一的框架理解了各种优化算法：

1. **优化问题的本质**：找到使损失函数最小的参数
2. **统一框架的三个组件**：
   - 动量 $v_t$：累积历史梯度，平滑更新
   - 自适应学习率 $s_t$：根据历史梯度调整学习率
   - 一阶矩估计 $m_t$：对梯度的平滑估计
3. **各算法的关系**：SGD → Momentum → AdaGrad → RMSProp → Adam → AdamW
4. **实践建议**：AdamW是当前最常用的优化器，配合warmup和学习率调度效果最佳

理解了优化算法，你就能更好地调参，让模型训练得更快、更稳定。下一篇我们将介绍**损失函数**，看看我们到底在优化什么。
