摘要

对比学习（Contrastive Learning）作为自监督学习的核心范式，通过构建正负样本对来学习具有判别性的特征表示，已成为计算机视觉领域的重要研究方向。这里参考李沐论文精读 ，系统梳理了2018年至2021年间CV领域对比学习的发展历程，涵盖14篇里程碑式论文，按技术演进划分为四个阶段：百花齐放期（2018-2019）、CV双雄期（2020）、无负样本探索期（2020-2021）和Transformer时代（2021）。报告详细分析了每篇论文的核心思想、关键技术创新、有效tricks及性能表现，并总结了技术演进脉络和未来发展方向。

1. 引言

1.1 对比学习的核心思想

对比学习是自监督学习（Self-supervised Learning，SSL）的重要代表，其基本思想是：将相似样本（正样本对）在特征空间中拉近，将不相似样本（负样本对）推远。这一简单而强大的思想为无监督/自监督学习提供了有效的训练信号。

对比学习通过代理任务（pretext task），来定义那些样本相似，那些不相似，从而产生监督信号来训练模型。代理任务通常是人为设定的规则，例如在计算机视觉中，对比学习通常通过以下方式构建正负样本对：

  • 正样本对：同一图像的不同数据增强视角（如随机裁剪、颜色抖动等）
  • 负样本对：不同图像之间的样本

1.2 对比学习的重要意义

  1. 突破标注瓶颈：无需大量人工标注即可学习高质量特征表示
  2. 性能逼近监督学习：在ImageNet等基准数据集上，自监督预训练的性能已接近甚至超越监督学习
  3. 迁移能力强：预训练模型在下游任务（目标检测、分割等）上表现优异
  4. 推动理论研究：促进了对深度学习表征学习本质的理解

1.3 发展历程概览

本报告将CV对比学习的发展划分为四个阶段：

阶段 时间 特征 代表工作
百花齐放 2018-2019 方法多样，性能有限 InstDisc, CPC, CMC
CV双雄 2020 MoCo与SimCLR竞争，性能飞跃 MoCo v1/v2, SimCLR v1/v2, SwAV
无负样本 2020-2021 突破负样本依赖 BYOL, SimSiam
Transformer 2021 架构升级，性能新高 MoCo v3, DINO

2. 第一阶段：百花齐放 (2018-2019)

在对比学习的萌芽期，研究者们探索了不同的代理任务、模型架构和目标函数，为后续发展奠定了基础。

2.1 InstDisc (CVPR 2018)

论文：*Unsupervised Feature Learning via Non-Parametric Instance Discrimination*

核心思想

提出”个体判别”（Instance Discrimination）任务：将每个图像实例视为一个独立的类别，通过区分不同实例来学习判别性特征。

关键技术创新

  1. Memory Bank机制

    • 存储整个数据集（128万张ImageNet图像）的特征向量
    • 解决了负样本数量受限于Batch Size的问题
    • 特征维度：128维，经过L2归一化

  2. NCE损失函数

    $$L_{NCE} = -E_{P_d}[\log h(i, v_{i}^{(t-1)})-\lambda|| v_{i}^{(t)}- v_{i}^{(t-1)}||2^2] - m \cdot E{P_n}[\log(1 - h(i, v^{‘(t-1)}))]$$

  3. Proximal Regularization
    因为每类样本只有一个实例，随机采样过程会带来大的波动，所以在损失函数里加了一个特征更新幅度的约束，使训练过程更稳定。

有效Tricks

  • 温度系数 $\tau = 0.07$
  • 负样本数量 $m = 4096$
  • 使用ResNet-50作为Encoder

性能表现

  • ImageNet线性评估 Top-1: 46.5%

历史意义

InstDisc首次将实例判别引入CV领域，开创了对比学习的先河，但Memory Bank的特征一致性问题为后续MoCo的改进留下了空间。

2.2 InvaSpread (CVPR 2019)

论文：*Unsupervised Embedding Learning via Invariant and Spreading Instance Feature*

核心思想

强调特征学习的两个关键属性：

  • 不变性（Invariance）：对数据增强保持鲁棒
  • 扩散性（Spreading）：不同实例在特征空间应尽可能分散

关键技术创新

  1. 端到端Siamese架构

    • 摒弃Memory Bank，实现纯端到端训练
    • 在同一mini-batch内构建正负样本对

  2. 基于多分类交叉熵的对比损失

    • 将对比学习转化为多分类问题
    • 每个样本的增强视角作为正类，batch内其他样本作为负类

局限性

  • 受限于当时的硬件条件，Batch Size仅为256
  • 负样本数量不足，限制了性能提升

性能表现

  • ImageNet线性评估 Top-1: 48.8%

历史意义

InvaSpread是SimCLR的重要前身，证明了端到端训练的可行性，为后续大Batch训练奠定了基础。

2.3 CPC v1 (2018)

论文：*Representation Learning with Contrastive Predictive Coding*

核心思想

通过预测序列的未来信息来学习表征，基于”能够预测未来的特征必然包含高层语义”的假设，是一种生成式的对比学习。

网络架构

  1. Encoder $g_{enc}$：将输入 $x_t$ 映射为潜在表示 $z_t$
  2. 自回归模型 $g_{ar}$（如GRU）：将历史信息 $z_{\le t}$ 总结为上下文向量 $c_t$
  3. 预测：使用 $c_t$ 预测未来的 $z_{t+k}$

InfoNCE损失函数

$$L_N = -\mathbb{E}X \left[ \log \frac{\exp(z{t+k}^T W_k c_t)}{\sum_{x_j \in X} \exp(z_j^T W_k c_t)} \right]$$

理论贡献：证明了最小化InfoNCE等价于最大化 $c_t$ 与 $x_{t+k}$ 之间的互信息下界。

在CV中的应用

  • 将图像切分为Patches
  • 利用空间位置关系进行预测（如预测下方或右侧的Patch）

性能表现

  • ImageNet线性评估 Top-1: 48.7%

历史意义

CPC提出的InfoNCE损失成为对比学习的标准损失函数，被后续大量工作采用。

2.4 CMC (ECCV 2019)

论文：*Contrastive Multiview Coding *

核心思想

多视角对比学习：人类通过多种感官（视觉、听觉、触觉）感知世界，不同视角共享相同的语义信息。

网络架构

  • 为每个View（如RGB、深度图、法线图）配备独立的Encoder
  • 将不同View的特征投影到同一个单位球面上

损失函数

扩展的InfoNCE损失，最大化同一物体不同View之间的互信息：

$$L = -\log \frac{\exp(f_1 \cdot f_2 / \tau)}{\sum_{j} \exp(f_1 \cdot f_j / \tau)}$$

其中 $f_1, f_2$ 是同一物体的不同View特征。

关键发现

  • View-invariant特征具有强大的泛化能力
  • 多视角学习优于单视角学习

历史意义

CMC将对比学习从单模态扩展到多模态，启发了后续多模态预训练工作（如CLIP）。这里使用了不同编码器来处理不同类型的输入，而后续transformer面世后，提供了统一处理不同模态数据的可能性。

第一阶段小结

论文 核心创新 负样本来源 性能 (Top-1)
InstDisc Memory Bank + NCE Loss Memory Bank 46.5%
InvaSpread 端到端Siamese Batch内 48.8%
CPC v1 InfoNCE + 预测未来 随机采样 48.7%
CMC 多视角对比 跨View -

阶段特征：方法多样化，但性能尚未突破50%，主要瓶颈在于负样本数量和特征一致性。

3. 第二阶段：CV双雄 (2020)

2020年是对比学习的爆发期，MoCo系列和SimCLR系列的竞争推动了技术快速迭代，性能实现飞跃。

3.1 MoCo v1 (CVPR 2020)

论文：*Momentum Contrast for Unsupervised Visual Representation Learning*

核心思想

将对比学习视为字典查询（Dictionary Look-up）任务，字典应满足：

  1. 规模大：包含足够多的负样本
  2. 一致性：特征在演进过程中保持一致

网络架构

  1. Query Encoder $f_q$：正常梯度更新
  2. Key Encoder $f_k$：动量更新
  3. Queue：存储最近的K个负样本特征

动量更新机制

$$\theta_k \leftarrow m \theta_k + (1-m) \theta_q$$

其中 $m = 0.999$ 为动量系数。

关键技术创新

  1. Queue机制

    • 解耦字典大小与Batch Size
    • Queue Size = 65536，远大于Batch Size (256)
    • 先进先出（FIFO）策略

  2. 动量更新

    • 保证Key Encoder缓慢演进，维持特征一致性
    • 避免Memory Bank的特征过时问题

损失函数

InfoNCE损失：

$$L_q = -\log \frac{\exp(q \cdot k_+ / \tau)}{ \sum_{i=0}^{K} \exp(q \cdot k_i / \tau)}$$

有效Tricks

  • Batch Size: 256
  • Queue Size: 65536
  • 动量系数: $m = 0.999$
  • 温度系数: $\tau = 0.07$

性能表现

  • ImageNet线性评估 Top-1: 68.6%

历史意义

MoCo首次将对比学习性能提升到60%以上，证明了动量机制的有效性，成为后续工作的重要基线。

3.2 SimCLR v1 (ICML 2020)

论文：*A Simple Framework for Contrastive Learning of Visual Representations*

核心思想

一个简单的端到端框架，通过强大的数据增强和非线性投影提升表征质量。

网络架构

  1. Encoder $f(\cdot)$：ResNet-50
  2. Projection Head $g(\cdot)$：2层MLP（关键创新）
    • 结构：Linear → ReLU → Linear
    • 输入维度：2048，输出维度：128

NT-Xent损失函数

Normalized Temperature-scaled Cross Entropy Loss:

$$l_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j)/\tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k)/\tau)}$$

其中 $\text{sim}(u, v) = u^T v / (|u| |v|)$ 为余弦相似度。

关键技术创新

  1. MLP Projection Head

    • 发现：Projector后的特征 $z$ 适合对比学习
    • 发现：Projector前的特征 $h$ 更适合下游任务
    • 这一发现至关重要，被后续工作广泛采用

  2. 强数据增强组合

    • RandomResizedCrop (0.08-1.0)
    • ColorJitter (strength=1.0)：亮度、对比度、饱和度、色调
    • GaussianBlur (10%概率)
    • RandomHorizontalFlip
      其中随机裁剪和随机颜色变换效果最好

  3. 大Batch训练

    • Batch Size: 4096（需要128个TPU核心）
    • 训练Epochs: 1000

性能表现

  • ImageNet线性评估 Top-1: 76.5%

历史意义

SimCLR证明了端到端训练的潜力，MLP Projector和强数据增强成为对比学习的标配。

3.3 MoCo v2 (2020)

论文：*Improved Baselines with Momentum Contrastive Learning*

核心思想

将SimCLR的有效组件集成到MoCo框架中，证明MoCo架构在小Batch下也能超越SimCLR。

改进点

  1. 增加MLP Projector（来自SimCLR）
  2. 增强数据增强（来自SimCLR）
    • 添加ColorJitter和GaussianBlur
  3. 使用余弦学习率调度

性能表现

  • ImageNet线性评估 Top-1: 71.1%
  • Batch Size仅256，可以在单机8卡下训练，性能超过需要更多机器的SimCLR v1

历史意义

MoCo v2证明了架构设计的重要性，小Batch + 好架构可以战胜大Batch + 简单架构。

3.4 SimCLR v2 (NeurIPS 2020)

论文：*Big Self-Supervised Models are Strong Semi-Supervised Learners*

核心思想

大模型是强半监督学习者，通过”自监督预训练 → 部分标签微调 → 知识蒸馏”三步走策略。

关键改进

  1. 更深的Projector

    • 从1层MLP增加到3层MLP(2层最优)
    • 中间层维度更大

  2. 更大的模型

    • ResNet-152 (3x)：宽度扩大3倍
    • 参数量大幅增加

  3. 半监督学习流程

    • Step 1: 自监督预训练（无标签）
    • Step 2: 使用1%或10%标签微调
    • Step 3: 将大模型知识蒸馏到小模型

  4. 动量编码器

    • 增加moco的动量编码器思路

性能表现

  • ImageNet 线性评估Top-1: 79.8%

历史意义

SimCLR v2证明了自监督预训练在半监督学习中的巨大潜力，为后续大模型预训练提供了思路。

3.5 CPC v2 (ICLR 2020)

论文：*Data-Efficient Image Recognition with Contrastive Predictive Coding*

核心思想

通过增大模型容量和改进训练细节提升CPC性能。

关键改进

  1. 更大的模型

    • 使用ResNet-161（161层）
    • 显著增加模型容量

  2. Layer Normalization替代Batch Normalization

    • 防止Patch之间的信息泄漏
    • BN会在batch维度上共享统计信息，导致作弊

  3. 多方向预测

    • 不仅预测下方和右侧，还预测上方和左侧
    • 增加预测任务的难度

性能表现

  • ImageNet线性评估 Top-1: 71.5%

历史意义

CPC v2证明了预测式对比学习的潜力，但复杂度较高，后续工作较少采用。

3.6 SwAV (NeurIPS 2020)

论文：*Unsupervised Learning of Visual Features by Contrasting Cluster Assignments*

核心思想

对比聚类指派（Cluster Assignments）而非特征本身，通过在线聚类实现无负样本对比学习。

网络架构

  1. Encoder $f(\cdot)$：ResNet-50
  2. Projection Head $g(\cdot)$：2层MLP
  3. Prototypes $C$：可学习的聚类中心（如3000个）

Swapped Prediction损失

给定同一图像的两个增强视角 $x_1, x_2$：

  1. 计算特征 $z_1 = g(f(x_1)), z_2 = g(f(x_2))$
  2. 使用Sinkhorn-Knopp算法计算聚类分配 $q_1, q_2$
  3. 交换预测：用 $z_1$ 预测 $q_2$，用 $z_2$ 预测 $q_1$

$$L(z_1, z_2) = \ell(z_1, q_2) + \ell(z_2, q_1)$$

关键技术创新

  1. Multi-crop策略

    • 2个标准分辨率裁剪（224×224）
    • 6个小分辨率裁剪（96×96）
    • 大幅增加正样本对数量（从1对增加到8对）

  2. 在线聚类

    • 使用Sinkhorn-Knopp算法快速求解最优传输问题
    • 避免聚类坍塌（所有样本分配到同一类）

性能表现

  • ImageNet线性评估 Top-1: 75.3%

历史意义

SwAV证明了聚类方法在对比学习中的有效性，Multi-crop策略被后续工作广泛采用。

第二阶段小结

论文 年份 核心创新 Batch Size 性能 (Top-1)
MoCo v1 2020 Momentum Queue 256 68.6%
SimCLR v1 2020 MLP Projector + 强增强 4096 76.5%
MoCo v2 2020 集成SimCLR tricks 256 71.1%
SimCLR v2 2020 大模型 + 半监督 4096 79.8%
CPC v2 2020 LayerNorm + 大模型 - 71.5%
SwAV 2020 在线聚类 + Multi-crop 4096 75.3%

阶段特征：

  • MoCo与SimCLR的竞争推动技术快速迭代
  • 性能从60%提升到75%，逼近监督学习
  • MLP Projector、强数据增强、Multi-crop成为标配

4. 第三阶段：不用负样本 (2020-2021)

这一阶段的研究挑战了”对比学习必须依赖负样本”的直觉，通过非对称结构避免模型坍塌。

4.1 BYOL (NeurIPS 2020)

论文：*Bootstrap Your Own Latent - A New Approach to Self-Supervised Learning*

核心思想

通过一个网络预测另一个网络的输出来学习，完全不需要负样本。

网络架构

  1. Online Network

    • Encoder $f_\theta$
    • Projector $g_\theta$
    • **Predictor $q_\theta$**（关键创新）

  2. Target Network

    • Encoder $f_\xi$
    • Projector $g_\xi$
    • 无Predictor

  3. 动量更新
    $$\xi \leftarrow \tau \xi + (1-\tau) \theta$$
    其中 $\tau = 0.996$

损失函数

均方误差（MSE）损失：

$$L_\theta = | \bar{q}_\theta(z_\theta) - \bar{z}’_\xi |^2_2$$

其中 $\bar{\cdot}$ 表示L2归一化。

为什么不坍塌？

  1. Predictor的非对称性：Online Network有Predictor，Target Network没有
  2. 动量更新：Target Network缓慢演进，提供稳定的目标
  3. Batch Normalization：隐式地引入了负样本信息（有争议）

性能表现

  • ImageNet线性评估 Top-1: 74.3%
  • 无需负样本，性能超过MoCo v2

历史意义

BYOL首次证明了无负样本对比学习的可行性，引发了对”为什么不坍塌”的广泛讨论。

4.2 SimSiam (CVPR 2021)

论文：*Exploring Simple Siamese Representation Learning*

核心思想

探究Siamese网络在没有负样本、没有动量编码器时为何不坍塌，发现Stop-gradient是核心。

网络架构

  1. 对称的Encoder $f(\cdot)$
  2. Projection Head $h(\cdot)$：3层MLP
  3. Prediction Head $g(\cdot)$：2层MLP（仅一侧有）

损失函数

负余弦相似度：

$$\mathcal{D}(p_1, z_2) = -\frac{p_1}{|p_1|_2} \cdot \frac{z_2}{|z_2|_2}$$

对称化损失：

$$\mathcal{L} = \frac{1}{2}\mathcal{D}(p_1, \text{stopgrad}(z_2)) + \frac{1}{2}\mathcal{D}(p_2, \text{stopgrad}(z_1))$$

关键发现：Stop-gradient是核心

消融实验：

配置 是否坍塌 性能
有Stop-gradient 否 71.3%
无Stop-gradient 是 0.1%
有动量更新，无Stop-gradient 是 0.1%

结论：

  • Stop-gradient是防止坍塌的充分必要条件
  • 动量更新不是必须的（与BYOL不同）
  • Predictor + Stop-gradient构成非对称性

理论解释

SimSiam可以看作EM算法的一种形式：

  • E-step：固定一侧网络（Stop-gradient），优化另一侧
  • M-step：交替优化

性能表现

  • ImageNet线性评估 Top-1: 71.3%
  • 架构最简单，但性能略低于BYOL

历史意义

SimSiam是对比学习的总结性工作，将所有方法归纳到统一框架中，揭示了防止坍塌的本质机制。

第三阶段小结

论文 负样本 动量更新 Predictor Stop-gradient 性能 (Top-1)
BYOL 否 是 是 隐式 74.3%
SimSiam 否 否 是 显式 71.3%

阶段特征：

  • 突破了”对比学习必须有负样本”的直觉
  • 揭示了非对称结构（Predictor + Stop-gradient）的重要性
  • 为后续简化对比学习框架提供了思路

5. 第四阶段：Transformer时代 (2021)

随着Vision Transformer (ViT)在CV领域的普及，对比学习也开始向Transformer架构迁移。

5.1 MoCo v3 (ICCV 2021)

论文：*An Empirical Study of Training Self-Supervised Vision Transformers*

核心思想

研究自监督ViT训练的不稳定性问题，并提出解决方案。

发现的问题

在将MoCo v2直接应用于ViT时，训练会出现：

  • 梯度突变（Gradient Spike）：损失突然增大
  • 训练不稳定：性能波动剧烈

问题定位

通过梯度分析发现：

  • 梯度突变主要发生在Patch Embedding层（第一层线性投影）
  • ViT的自注意力机制放大了第一层的梯度

解决方案：Fixed Random Patch Projection

冻结第一层线性投影层，不参与训练：

其他改进

  1. 使用AdamW优化器（而非SGD）
  2. 更大的Batch Size：4096
  3. 更长的训练：300 epochs

性能表现

  • ImageNet线性评估 Top-1: 81%

历史意义

MoCo v3首次成功将对比学习应用于ViT，性能突破80%，为后续ViT自监督学习奠定了基础。

5.2 DINO (ICCV 2021)

论文：*Emerging Properties in Self-Supervised Vision Transformers*

核心思想

自蒸馏（Self-Distillation）框架，发现自监督ViT能自动学习到物体的语义分割边界。

网络架构

  1. Student Network：正常梯度更新
  2. Teacher Network：动量更新（EMA）
    $$\theta_t \leftarrow \lambda \theta_t + (1-\lambda) \theta_s$$
    其中 $\lambda$ 从0.996线性增加到1

损失函数

交叉熵损失，匹配Student和Teacher的输出分布：

$$\min_{\theta_s} H(P_t(x), P_s(x))$$

其中 $P$ 是经过softmax的输出分布。

关键技术创新

  1. Centering操作

    • 维护Teacher输出的指数移动平均 $c$
    • 在softmax前减去 $c$，防止某一维度主导

    $$P_t(x) = \text{softmax}\left(\frac{g_{\theta_t}(x) - c}{\tau_t}\right)$$

  2. Sharpening操作

    • Student使用较低的温度 $\tau_s = 0.1$
    • Teacher使用较高的温度 $\tau_t = 0.04 \to 0.07$（warmup）

  3. Multi-crop策略

    • 2个全局视角（224×224）
    • 多个局部视角（96×96）
    • 仅全局视角通过Teacher

惊人发现：自动学习语义分割

无需任何分割标注，DINO的ViT自注意力图能够：

  • 自动分割出前景物体
  • 捕捉物体的精细边界
  • 对不同物体类别泛化

这一现象在CNN中从未观察到，是ViT的独特性质。

性能表现

  • ImageNet线性评估 Top-1: 80.1%

有效Tricks

  • 小Patch Size（8×8）显著提升性能
  • Multi-crop增强
  • Centering + Sharpening防止坍塌

历史意义

DINO不仅在性能上达到SOTA，更重要的是发现了自监督ViT的涌现特性（Emerging Properties），为理解ViT提供了新视角。

第四阶段小结

论文 架构 核心创新 性能 (Top-1)
MoCo v3 ViT Fixed Patch Projection 81.0%
DINO ViT Centering + 自蒸馏 80.1%

阶段特征：

  • 成功将对比学习迁移到ViT架构
  • 性能突破80%，达到新高度
  • 发现ViT的独特性质（自动语义分割）

8. 技术演进脉络深度分析

8.1 负样本策略的演进

阶段1：Memory Bank（InstDisc）

  • 优点：负样本数量不受Batch Size限制
  • 缺点：特征不一致（更新滞后）

阶段2：Queue + Momentum（MoCo）

  • 优点：特征一致性好，负样本数量大
  • 缺点：需要额外的动量编码器

阶段3：大Batch（SimCLR）

  • 优点：端到端训练，特征完全一致
  • 缺点：需要大量计算资源

阶段4：无需负样本（BYOL/SimSiam）

  • 优点：简化训练，无需大Batch或Queue
  • 缺点：需要精心设计防止坍塌

8.2 防止坍塌的机制

坍塌问题：所有样本映射到相同的特征，模型失去判别能力。

解决方案演进：

  1. 负样本推开（InstDisc, MoCo, SimCLR）

    • 通过对比损失显式推开负样本
    • 需要大量负样本

  2. 聚类约束（SwAV）

    • 通过聚类分配约束特征分布
    • 使用Sinkhorn-Knopp算法保证均匀分配

  3. 非对称架构（BYOL, SimSiam）

    • Predictor打破对称性
    • Stop-gradient阻止信息流

  4. Centering（DINO）

    • 减去输出均值，防止某一维度主导
    • 结合Sharpening控制熵

8.3 数据增强的重要性

SimCLR的消融实验表明，数据增强对性能的影响巨大：

增强组合 性能 (Top-1)
仅RandomCrop 48.3%
+ ColorJitter 61.9% (+13.6%)
+ GaussianBlur 66.6% (+4.7%)

关键发现：

  • ColorJitter是最重要的增强（+13.6%）
  • 不同增强的组合效果优于单一增强
  • 增强强度需要精心调节

8.4 架构演进：CNN → ViT

CNN时代的挑战：

  • 归纳偏置强（局部性、平移不变性）
  • 难以捕捉全局依赖

ViT的优势：

  • 自注意力机制捕捉全局信息
  • 更灵活的表征能力

ViT的挑战：

  • 训练不稳定（梯度突变）
  • 需要更多数据

解决方案：

  • MoCo v3：Fixed Patch Projection
  • DINO：Centering + 自蒸馏

9. 总结与展望

9.1 发展规律总结

  1. 从复杂到简单

    • 早期：Memory Bank、复杂的代理任务
    • 后期：简单的Siamese架构 + Stop-gradient

  2. 从依赖负样本到无需负样本

    • 早期：大量负样本是必须的
    • 后期：通过非对称结构避免坍塌

  3. 从小模型到大模型

    • 早期：ResNet-50
    • 后期：ResNet-152 (3x)、ViT-L

  4. 从CNN到Transformer

    • 2018-2020：CNN主导
    • 2021-：ViT成为主流

9.2 核心经验总结

必备组件：

  1. MLP Projector：分离对比空间与表征空间
  2. 强数据增强：ColorJitter + GaussianBlur
  3. 防坍塌机制：负样本 / Predictor / Centering

有效Tricks：

  1. Multi-crop：增加正样本对，提升性能
  2. 动量更新：稳定特征演进
  3. 温度系数：控制分布的平滑度（通常0.07-0.1）
  4. 长训练：800-1000 epochs

9.3 未来发展方向

1. 多模态对比学习

  • CLIP（2021）：图像-文本对比学习
  • ALIGN（2021）：大规模噪声数据训练
  • 方向：视觉-语言-音频多模态统一表征

2. 大规模预训练

  • MAE（2021）：Masked Autoencoder
  • BEiT（2021）：BERT式的视觉预训练
  • 方向：结合对比学习与生成式学习
